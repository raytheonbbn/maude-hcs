cp2-exe-scenarios contains actor definitions and
several initial configurations parameterized the list of files
to exfil.   

iS(fl:ByteSeqL)  --- weird network: alice,bob, fakedns/raceboat tunnels
                 --- the fake dns tunnel the iodine client send queries
                 --- directly to the iodine server
                 
iSR(fl:ByteSeqL)  ---- weird network: alice,bob, dns/raceboat tunnels

dnsTgConf --- dns generated traffic only
masTgConf --- mastodon generated traffic only
iSdnsTg(fl:ByteSeqL)    --- weird net and dns generated traffic
iSmasTg(fl:ByteSeqL)   --- weird net and mastodon generated traffic
iSdmsmasTg(fl:ByteSeqL) --- weird net, nastodon,dns generated traffic

4 files, F0 F1 F2 F3, of increasing size are defined.


All configurations with background traffic include the adversary.
See the end of the file for commands to run each scenario.

cp2-exe-scenarios loads dnsnet.maude and cp2-e2e-shared.maude
(sload ./dsnet, ./cp2-e2e-shared) also attached.   
The sloads assume these files are in some directory two levels
below lib -- in my version its lib/test-clt/maude
