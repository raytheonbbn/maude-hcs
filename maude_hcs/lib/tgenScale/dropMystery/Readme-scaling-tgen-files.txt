Equational tgen actor specifiation files

------ modules common to all tgen-eq systems in common/
ua-actor-eq.maude 
  -- equational version of user-model actor
  -- parameterized by a stop time 
    -- in CP2 the time that alice sets the doneFlag
    -- in CP3 -- TBD
  -- this is in addition to the nsteps termination condition

tgen-eq-aux.maude  --- integration with hcs execution
  --- defines the finalize hook into the scheduler to run an
      equational tgen configuration after the hcs scenario
      completes
  --- finalize assumes 
      * a function tgenConfig that constructs
        the tgen initial configuration
      * a function augmentAdvAttrs that can be used to add
        summary information about the adversary attributes
        for example binning subsets of timestamped message lists
          
analysis-aux.maude
  --- defines functions for filtering and binning TimeStampMsgList
        
------------------- dns tgen files in dns-eq/
see dns2dns-eq.txt for description of dns server network 

dnsQgen-actor-eq.maude 
  --- equational version of the dnsQgen 
     (query generator) actor 
     
dnsObs-actor-eq.maude  
  --- observer pattern instantiated to DNS
  --- sits between the dns client and its resolver
  --- logs query with sendtime in tgenSent:
  --- logs responses with receivetime in tgenRcvd:

dnsClient-actor-eq.maude
  --- transforms names generated by dnsQgen to dns queries
  --- uses attributes retryTO:, retryMax:, nRetry: attributes
       to manage retry in case of dropped / delayed response

dnsTgen-eq.maude loads the dns-eq and the tgen -eq actors
defines  DNSTGEN-EQ that includes all these actors.
